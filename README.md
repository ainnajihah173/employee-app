# Employee Web Based System

A Laravel-based employee management application for tracking employee profiles, departments, roles, and status with role-protected HR administration.

## Features

- User authentication and role-based access control
- HR admin protected employee creation, update, and deletion
- Employee listing, search, detail view, and pagination
- Auto-generated department-based employee ID
- Dashboard with total employees, department statistics, and recent hires

## Technology Stack

- PHP 8.3+
- Laravel 13.x
- MySQL / MariaDB (or compatible SQL database)
- Laravel Breeze / built-in auth
- Tailwind CSS
- Vite
- Alpine.js

## Requirements

- PHP 8.3 or newer
- Composer
- Node.js and npm
- Database server (MySQL, MariaDB, PostgreSQL, SQLite)
- Git (optional)

## Step-by-step Setup

1. Clone the repository:

   ```bash
   git clone <repository-url> employee-app
   cd employee-app
   ```

2. Install PHP dependencies:

   ```bash
   composer install
   ```

3. Copy the example environment file:

   ```bash
   copy .env.example .env
   ```

4. Configure your database connection in `.env`:

   ```ini
   APP_NAME="Employee App"
   APP_ENV=local
   APP_KEY=
   APP_DEBUG=true
   APP_URL=http://localhost

   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=employee_app
   DB_USERNAME=root
   DB_PASSWORD=
   ```

   Adjust values for your local environment.

5. Generate the application key:

   ```bash
   php artisan key:generate
   ```

6. Run database migrations:

   ```bash
   php artisan migrate
   ```

7. Install front-end dependencies:

   ```bash
   npm install
   ```

8. Build assets for development or production:

   ```bash
   npm run build
   ```

## Run the Application

- Start the Laravel development server:

  ```bash
  php artisan serve
  ```

- Alternatively, run the Vite dev server:

  ```bash
  npm run dev
  ```

Then open `http://127.0.0.1:8000` or the Vite URL in your browser.

## Database Design

### `users`

The main authentication and authorization table.

| Column             | Type       | Notes |
|-------------------|------------|-------|
| `id`              | bigint     | Primary key |
| `email`           | string     | Unique email login |
| `email_verified_at`| timestamp | Nullable |
| `status`          | string     | Default `Active` |
| `password`        | string     | Hashed password |
| `role`            | string     | `employee` or `hr_admin` |
| `remember_token`  | string     | Nullable for auth sessions |
| `created_at`      | timestamp  | Laravel timestamps |
| `updated_at`      | timestamp  | Laravel timestamps |

### `employees`

Employee profile records linked to the user account.

| Column        | Type       | Notes |
|--------------|------------|-------|
| `id`         | bigint     | Primary key |
| `user_id`    | bigint     | Nullable foreign key to `users.id`, cascade on delete |
| `employee_id`| string     | Unique generated ID (e.g. `HR-0001`) |
| `name`       | string     | Employee full name |
| `department` | string     | Department name |
| `gender`     | enum       | `Male`, `Female`|
| `created_at` | timestamp  | Laravel timestamps |
| `updated_at` | timestamp  | Laravel timestamps |

### Relationship

- `User` has one `Employee`
- `Employee` belongs to `User`

### Employee ID Generation

Employee IDs are auto-generated by department prefix and an incremental number, e.g.:

- `Information Technology` → `IT-0001`
- `Human Resource` → `HR-0001`
- `Finance` → `FIN-0001`

## Application Flow

- All routes are protected by `auth` middleware.
- HR admin actions are protected by the `hr.admin` middleware.
- The employee controller allows search by name, employee ID, and department.
- The dashboard displays total employee count, department breakdown, and latest hires.
